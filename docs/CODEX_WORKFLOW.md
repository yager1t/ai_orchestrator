# Рабочий процесс в Codex

## 1. Первый запуск

```bash
cd ai-orchestrator
codex
```

Первый prompt:

```text
Прочитай AGENTS.md, docs/AGENT_TASK_DISTRIBUTION.md и tasks/001_mvp_bootstrap.md.
Работай в режиме PROPOSE: предложи первый bounded step без изменения файлов.
```

---

## 2. Запуск на реализацию

```text
Режим APPLY.
Выполни только Step 1 из tasks/001_mvp_bootstrap.md:
реализуй AgentAdapter и MockAgentAdapter.
Не меняй другие модули.
После изменения запусти python -m pytest.
```

---

## 3. Запуск проверки

```text
Режим VERIFY.
Проверь текущий diff, запусти:
python -m compileall ai_orchestrator
python -m pytest
git diff --check
Верни отчёт по формату из AGENTS.md.
```

---

## 4. Если тесты упали

```text
Режим REPAIR.
Исправь только ошибки из последнего вывода pytest.
Не добавляй новую функциональность.
После исправления снова запусти pytest.
```

---

## 5. Если нужен review

```text
Режим REVIEW.
Проверь текущий diff как Review Agent.
Найди:
1. нарушения архитектуры;
2. scope creep;
3. отсутствие тестов;
4. риски безопасности;
5. проблемы документации.
Код не меняй.
```

---

## 6. Неинтерактивный запуск Codex

Пример:

```bash
codex exec --sandbox workspace-write \
  "Прочитай AGENTS.md и tasks/001_mvp_bootstrap.md. Выполни только первый bounded step, затем запусти проверки."
```

Для машинного вывода:

```bash
codex exec --json \
  "Сделай review текущего diff и верни краткий отчёт"
```

---

## 7. Правило остановки

Остановить итерацию и запросить решение пользователя, если:

- нужно добавить крупную зависимость;
- нужно изменить архитектурную границу;
- нужно выполнить опасную команду;
- неясен DoD;
- задача требует доступа к секретам;
- изменения затрагивают более 8 файлов без явной причины.
