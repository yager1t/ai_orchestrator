# Правила разработки проекта с помощью ИИ

## 1. Базовый принцип

ИИ-агент не является единственным источником истины. Он может писать код, но готовность подтверждается только:

- тестами;
- проверками;
- diff-review;
- соответствием Definition of Done;
- отсутствием нарушений policy.

---

## 2. Режимы работы

### EXPLAIN

Только объяснить. Ничего не менять.

### PROPOSE

Предложить план и список файлов. Ничего не менять.

### APPLY

Внести изменения.

### VERIFY

Запустить проверки.

### REPAIR

Исправить найденные ошибки.

### REVIEW

Проверить diff, риски, безопасность.

---

## 3. Правило одного изменения

Одна итерация = один bounded change.

Плохо:

> “Реализуй весь MVP целиком”.

Хорошо:

> “Реализуй `AgentAdapter` и `MockAgentAdapter`, добавь тесты”.

---

## 4. Обязательный цикл

```text
1. Прочитать задачу.
2. Определить роль агента.
3. Сформулировать DoD.
4. Назвать файлы, которые будут изменены.
5. Сделать минимальный diff.
6. Запустить проверки.
7. Исправить ошибки.
8. Обновить документацию.
9. Зафиксировать итог.
```

---

## 5. Запрет на бесконтрольное расширение scope

Агент не должен:

- добавлять новые фреймворки без причины;
- переписывать модули, которые не относятся к задаче;
- менять архитектуру “заодно”;
- удалять старый код без миграционного плана;
- оставлять TODO вместо реализации, если задача требовала рабочий результат.

---

## 6. Проверки

Минимальные проверки:

```bash
python -m pytest
python -m compileall ai_orchestrator
git diff --check
```

Дополнительные проверки после подключения инструментов:

```bash
ruff check .
ruff format --check .
mypy ai_orchestrator
```

---

## 7. Документирование решений

Все архитектурные решения писать в `docs/DECISIONS.md`.

Формат:

```markdown
## ADR-0001: Название

Дата: YYYY-MM-DD

### Контекст

### Решение

### Последствия

### Альтернативы
```

---

## 8. Отчёт после каждой итерации

Формат ответа агента:

```markdown
## Что сделано

## Изменённые файлы

## Проверки

## Риски

## Следующий шаг
```

---

## 9. Правило честности

Если проверка не запускалась, нельзя писать “тесты проходят”.

Правильно:

> `python -m pytest` не запускался, потому что зависимости ещё не установлены.

Неправильно:

> Всё должно работать.
